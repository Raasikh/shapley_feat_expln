# SHAP-Based Explainability for Regulated ML Compliance

## What This Is

A runnable demo that backs the Hive AI resume bullet:
> *"Compliance approval improved 22% by integrating SHAP-based explainability layers for regulated ML outputs"*

Builds a complete explainability pipeline from scratch: synthetic regulated data → XGBoost model → SHAP explanation layer → simulated compliance A/B test → explanation quality monitoring.

## Quick Start

```bash
# Upload SHAP_Compliance_Explainability.ipynb to Google Colab
# Runtime → CPU (no GPU needed)
# Run All Cells → ~3 minutes
```

**Dependencies** (auto-installed in Colab):
```
shap, xgboost, scikit-learn, matplotlib, pandas, numpy, tabulate
```

## What the Notebook Builds

### 1. Synthetic Regulated Dataset (15K records)
Simulates content moderation / risk scoring with known causal structure:
- 14 features: behavioral, content, historical, and data quality signals
- ~20% flag rate with realistic class imbalance
- Known causal relationships so you can verify SHAP explanations are correct

### 2. XGBoost Classifier + SHAP TreeExplainer
- Trains model with early stopping
- Initializes exact TreeExplainer (polynomial time for tree models)
- Explains expected_value as the baseline SHAP pushes from

### 3. ExplainabilityLayer Class (THE RESUME BULLET)
This is the core component — sits between model inference and compliance queue:
- `explain_single()`: per-prediction SHAP explanation with ranked feature contributions
- `explain_batch()`: batch processing for the compliance review queue
- Adds percentile context, confidence tiers, human-readable text
- Packages everything as a structured payload for reviewers

### 4. SHAP Visualizations
- Global feature importance bar chart
- Beeswarm plot showing impact direction
- Waterfall plot (exactly what a reviewer sees)
- Verification against known causal structure

### 5. Simulated Compliance A/B Test
- Control: reviewer sees prediction only
- Treatment: reviewer sees prediction + SHAP explanation
- Models reviewer behavior: high/medium/low confidence tiers
- SHAP helps most on medium-confidence predictions
- Measures first-pass approval rate and review time

### 6. Explanation Quality Monitoring
- Feature importance drift detection (rank correlation)
- Data quality feature exposure tracking
- Retraining trigger logic (connects to your retraining pipeline bullet)

### 7. SHAP Latency Benchmark
- TreeExplainer vs KernelExplainer head-to-head
- Per-record and batch latency measurements
- Shows why TreeExplainer was chosen for tree models

## Expected Results

| Metric | No SHAP | With SHAP | Change |
|--------|---------|-----------|--------|
| First-Pass Approval | ~57-62% | ~70-75% | **+20-25%** |
| Median Review Time | ~9-11 min | ~5-7 min | **-35-45%** |
| TreeExplainer Latency | — | ~0.5-2 ms/record | Fast enough |
| KernelExplainer Latency | — | ~200-800 ms/record | 100-500x slower |

The exact 22% will vary by random seed, but the A/B structure consistently shows ~20-25% improvement.
